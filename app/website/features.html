Why use Terraform + S3 + CloudFront?

This page highlights the benefits of automating a static website deployment using Terraform, Amazon S3, and CloudFront.

Terraform (Infrastructure as Code)

  • Repeatable: Declare infrastructure once and recreate consistently across environments.
  • Versioned: Store changes in Git to review history and roll back if needed.
  • Idempotent: Applying the same configuration produces the same result.
  • Automatable: Integrates with CI/CD to provision and publish automatically.

Amazon S3 (Static Site Hosting)

  • Durable & Scalable: High durability and built for serving large numbers of objects.
  • Cost-effective: Pay for storage and requests, cheap for static assets.
  • Simple: Store files like `index.html` and `features.html` and serve them via HTTP.

CloudFront (Global Delivery)

  • Fast: Edge caching reduces latency for users worldwide.
  • Secure: TLS termination and integrated access controls (OAC/OAI) to protect S3 buckets.
  • Reliable: Built-in redundancy and regional failover.

Putting them together: Terraform creates the S3 bucket and CloudFront distribution, then uploads your static files. CloudFront serves those files globally and caches them for performance.
